Software and Skills

Linux - When I was about 10 years old, my dad brought my first desktop computer home from work. It didn't have a soundcard, and I broke it almost instantly by going administrator mode and deleting critical files. It wasn't long before I began using Ubuntu at a very basic level. At around 20 I switched from Ubuntu to Arch Linux, and very recently I've switched to NixOS as it best enables an infrastructure-as-code approach at the OS level.

Shell Scripting - I've never had a problem writing shell scripts, but every time I read someone else's code I wonder if I don't really understand it.

Votebuilder / VAN / NGP - I know it in and out. Been working with the software since I started my career.

SQL - I consider myself to basically be 'done' with learning SQL. BigQuery has a number of additions that I continue to discover and find useful, but I can take cleaned data and construct a schema with only a little brainstorming.

R - Probably the single language I know best, especially the tidyverse ecosystem. I use it as an exploratory language, like shell scripting for data science.

Python - I wouldn't say I know Python 'well', but I can do basic plumbing with it. For situations where you have a RESTful API and need to regularly load data into SQL, it's about the perfect choice. For nearly anything else, I use something higher (R or Shell Scripting) or lower (Haskell) level.

Haskell - C for math majors and people who like the functional programming over procedural. Learning it has been incredibly rewarding, but I wouldn't say it's a particularly useful language.

QGIS - I feel like I've done a thousand different things in QGIS and haven't even scratched the surface. Every time I approach it with a new project I end up exploring a new feature or plugin.

Tableau / Looker Studio - Grouping these together to say that I know how to dashboard, and understand the benefits of dashboard-mindset when it comes to data communication.
